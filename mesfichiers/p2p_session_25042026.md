Les utilisateurs iront au kiosque Yabisso pour recharger leur contenu (data, packs, mises à jour).

---

## Kiosques Offline (Source Principale)

### Principe:
- Kiosque vend data en offline
- Pack compressé de taille ILLIMITÉE (jusqu'à 10GB+)
- Utilisateur recharge son contenu au kiosque

### Types de Recharge:
1. **Recharge Loba** - Nouvelles vidéos, images, stories
2. **Recharge Marché** - Nouveaux produits
3. **Recharge Hôtel** - Nouvelles chambres/offres
4. **Recharge Complète** - Tout le contenu Yabisso

### Étapes au Kiosque:
1. Utilisateur se rend au kiosque Yabisso
2. Scanner QR code ou proximity detection
3. Kiosque propose les packs disponibles:
   - Pack Loba 5GB
   - Pack Marché 2GB
   - Pack Hôtel 1GB
   - Pack Complet 10GB
4. Paiement (cash/mobile money)
5. Transfert Pack → Téléphone via WiFi Direct
6. Décompression et stockage automatique

---

## P2P Entre Utilisateurs (Source Secondaire)

### Principe:
- Complément au kiosque
- Échange entre utilisateurs pour partager du contenu manquant
- Limite: 200MB par service par jour entre chaque pair

### Cas d'usage:
- Utilisateur A a du contenu que B n'a pas
- B peut demander à A de partager
- Pas besoin d'aller au kiosque pour petit échange

### Limites:
```
A ↔ B = 200MB/service/jour
A ↔ C = 200MB/service/jour
B ↔ C = 200MB/service/jour
```

## Session 27 Avril 2026 - Fix Partage P2P Entre Users

### Problème:
- Le partage entre users utilise le pack ZIP (comme pour Kiosque)
- Pack ZIP = illimité mais uniquement Kiosque -> User
- P2P Entre Users = envoi direct SANS pack ZIP, limite 200MB/jour/service

### Solution Implémentée:
1. **P2PAutoSync._p2pSyncCycle()** - Changé pour envoi direct sans pack ZIP
2. **_sendDirectLobaContent()** - Envoie chaque fichier individuellement (max 200MB/jour)
3. **metadata.action = 'FILE_TRANSFER'** - Pour que le récepteur détecte le fichier
4. **Debug logging** - Pour tracer les erreurs

### Flux corrigé:
```
User A (Envoyeur)                    User B (Récepteur)
    |                                    |
    |-- metadata {action:FILE_TRANSFER}--> |
    |-------- Fichier direct ----------->|
    |                                    |
    |    (WifiDirectService.receiveFile)  |
    |                                    |
    |        _handleReceivedFile()        |
    |           ↓                        |
    |    Insert DB loba_posts            |
    |           ↓                        |
    |    onSyncStatus: SUCCESS          |
    |           ↓                        |
    |    LobaHomeScreen refresh         |
```

### Modifications:
- `app/src/features/bluetooth/services/P2PAutoSync.js`
  - `_p2pSyncCycle()`: Envoi direct sans pack
  - `_sendDirectLobaContent()`: Nouvelle méthode (200MB limit)
  - `_handleReceivedFile()`: Debug logging ajouté
